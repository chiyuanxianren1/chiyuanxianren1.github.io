rust简介

Rust 是一种系统编程语言，它专注于安全、并发和性能。Rust 由 Mozilla 研究院开发，并在 2010 年首次发布。以下是 Rust 语言的一些关键特点：

  1. 内存安全：Rust 通过所有权（ownership）、借用（borrowing）和生命周期（lifetimes）的概念，无需垃圾回收就能保证内存安全。

  2. 类型安全：Rust 是一种静态类型语言，它在编译时强制类型检查，减少了运行时错误。

  3. 并发编程：Rust 提供了强大的并发编程模型，允许开发者编写无数据竞争和无死锁的代码。

  4. 性能：Rust 编译成机器码，能够生成高性能的代码，接近于 C 和 C++ 的性能。

  5. 跨平台：Rust 支持多种操作系统和架构，包括 Windows、macOS、Linux 以及各种嵌入式系统。

  6. 包管理：Rust 有一个强大的包管理器和构建工具，叫做 Cargo，它简化了依赖管理和构建过程。

  7. 宏系统：Rust 提供了强大的宏系统，允许代码生成和元编程。

  8. 错误处理：Rust 使用 Result 和 Option 类型来处理可能的错误，鼓励显式的错误处理。

  9. 模式匹配：Rust 的模式匹配语法使得处理不同情况的代码更加简洁和清晰。

  10. 社区驱动：Rust 有一个活跃和友好的社区，社区成员贡献了大量的库和工具。

  11. 可扩展性：Rust 的语法和标准库都是可扩展的，允许开发者根据需要添加新的功能。

  12. 无全局异常：Rust 没有全局异常机制，而是使用 panic! 宏来处理异常情况，这有助于避免隐藏的控制流和不可预测的行为。

Rust 语言因其在安全性和性能方面的平衡而受到许多系统级开发者的青睐，尤其是在需要高性能和高可靠性的领域，如操作系统、嵌入式系统、网络编程和游戏开发等。

类似指针，但是比指针安全。
将字符串的引用作为参数传递给函数，而不是复制，这应该可以避免所有权的问题。引用类型是没有所有权的。


rust的可变引用和不可变引用的并发冲突问题是针对同一个变量的。可以同时读，但读写不共存，也不能同时写，保证数据同步，这一点能看出rust的设计思想，将我们在并发编程中可能遇到的问题，在平日的代码中就设计到，让我们时刻在思考数据的安全性。

两种字符串类型

&str和String.很明显看出&str是不可变引用类型，想要变就必须转为String类型。

Option

Option是一个枚举类型。虽然Rust中有null的概念，但是使用null并不是Rust中常见的模式。rust中一般采用Option中的None作为空。但采用此方式相对的会使获取数据等操作变得麻烦。非空的数据是由Option的另一个变量Some封装的。需要调用unwrap()才能解包。这也能解释为什么rust中的Hash Map获取数据后需要unwrap()才能获取真实数据。因为HashMap就是采用了Option。

注意ref和&的区别

ref是把数据复制到栈上，没有所有权转换的问题。&属于一种类型，如果形参使用&，那么实参就必须传&，而ref不是这样的。

lifetime生命周期

基础知识

  1. 函数生命周期
  • 原因：编译器有时无法静态推断函数中形参和返回值的生命周期，需要显式声明参数的生命周期。
  • 目标：防止悬空指针
  • 规则：同名生命周期参数绑定的对象
  1. struct生命周期
    当一个生命周期参数修饰多个字段的时候，编译器会将这个生命周期参数推断出这几个字段生命周期最小的那个。

其他的智能指针（太多了好难理解）

  • Rc（引用计数）
    是 Rust 中的一种类型，用于在单线程环境下共享同一份数据。当需要多个所有权时，可以使用 Rc，它跟踪引用的数量，允许多个所有者共享值，而不需要考虑线程安全问题。

  • Arc（原子引用计数）
    是一种智能指针类型，用于提供线程安全的共享所有权机制。它允许多个线程同时持有同一个数据结构的访问权，并在所有持有者释放对数据的引用时自动释放底层资源。
    Rc与Arc的用法差不多。

  • Cow智能指针
    （做题之后还是一脸懵，返回值的Owned和Borrowed到底有什么区别啊！）

    • 读写分离：在一些业务场景中，需要对某个数据结构进行多次读取和少量修改，但是每次修改都会导致内存分配和复制操作，从而影响程序的性能和效率。 Cow 类型可以通过克隆操作来避免这个问题，从而提高程序的性能和效率。
    • 借用检查：在 Rust 中，借用检查是一项重要的安全特性，可以避免程序中出现内存安全问题。 但是，在某些情况下，借用检查会导致代码的复杂度和可读性变差。 Cow 类型可以通过引用和克隆操作来解决这个问题，从而简化代码的实现和维护。

  • 实现类型转换From

    这段代码的逻辑如下：

    1. 使用split_once方法尝试将输入的字符串s在第一个逗号处分割成name和age两部分。
    2. 如果分割成功，去除name和age两端的空白字符，并将它们转换为String类型。
    3. 如果分割失败，返回Person的默认实例。
    4. 尝试将age字符串解析为usize类型。
    5. 如果解析成功且name不为空，创建并返回一个Person实例，其中包含解析后的name和age。
    6. 如果name为空或age解析失败，返回Person的默认实例。

  • 实现类型转换FromStr

    这段代码的逻辑如下：

  1. 定义了FromStr trait的关联错误类型Err为ParsePersonError。
  2. 实现了from_str函数，它接受一个字符串切片&str并返回一个Result，其中包含Person实例或解析错误。
  3. 首先检查输入字符串是否为空，如果为空，则返回ParsePersonError::Empty错误。
  4. 使用split方法将字符串按逗号分割，并收集结果到一个Vec<&str>向量中。
  5. 使用模式匹配来获取name和age。如果分割后的向量长度不正确，返回ParsePersonError::BadLen错误。
  6. 尝试将age部分解析为整数，如果解析失败，使用map_err将错误转换为ParsePersonError::ParseInt。
  7. 检查name是否为空，如果为空，返回ParsePersonError::NoName错误。
  8. 如果所有检查都通过，创建并返回一个Person实例。

在项目中加深对裸指针的理解

&mut t获取到变量t的引用（包含内存地址），把t转为mut u32裸指针（真实的32位内存地址），再转为usize类型的变量（已经不是裸指针了）。所以在函数体中若想通过解引用裸指针来访问t，就需要先把address转为裸指针类型，在通过的方式解引用来改变t的值。
